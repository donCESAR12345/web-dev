Networking tools in Linux CLI

In this tutorial you will presented to some of the most used networking commands in Linux, these commands are useful to know if a host is up and receiving requests, or to scan a network and find pingable hosts, trace the rout of a ping packet over a network. As you may know, Linux is well known for being a versatile tool in the computing area, and of course, this includes networking them. Base Linux distributions include at least the first command of this list, if you want to be a system administrator or network engineer (among many other careers) you must know at least the existence of the commands, because you can always check the manual (man).

ip or ifconfig

The ip command is the newer version of the well known ifconfig command, which is deprecated on most Linux machines, it is used to show and/or manipulate routing, network devices, interfaces and tunnels, as always, you can use ip -h to show a not so detailed help on the command usage, or if you really want to read and understand the command, use the man command giving "ping" as the argument.

netstat or ss

The ss command is the newer version of the netstat command, which is deprecated as well, its purpose is to display sockets statistics (as it name implies), revealing what applications are communicating to the internet or even between them locally in your computer. Most of the options are filters, to tell the command which kind of sockets you want to list or how you want to see them in the case you will parse them later.

curl

The curl command is a really powerful tool to send petitions to servers, including several protocols like HTTP, HTTPS, SSH, FTP and SMB. Commonly used to download files from servers.

nmap

The nmap command is a complicated mess, so I'll just cite its manual:

Nmap is an open source tool for network exploration. It was designed to rapidly scan large networks. Nmap uses raw IP packets to determine what hosts are available on the network, what services (application name and version) those hosts are offering, what operating systems (and OS versions) they are running, what type of packet filters/firewalls are in use, and dozens of other characteristics. While Nmap is commonly used for security audits, many systems and network administrators find it useful for routine tasks such as network inventory, managing service upgrade schedules, and monitoring host or service uptime.
